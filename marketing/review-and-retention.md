# Reviews and Retention

## Review Prompt Timing

Ask for reviews only after positive, earned moments:

- first room clear
- first loot card
- first Depth I completion
- first boss win
- first high-XP bank

Never prompt after:

- death
- void backlash
- crash recovery
- first failed run
- app launch before play

## In-App Review Copy

`Enjoying the descent? A quick rating helps the dungeon reach more players.`

Keep the prompt optional and easy to dismiss.

## Negative Review Replies

Bug report:

`Thanks for calling this out. I’m tracking it for the next patch. If you can share your device model and what happened right before the issue, that will help me fix it faster.`

Balance complaint:

`Fair feedback. The press mechanic should feel risky, not unfair. I’m reviewing early-room tuning and appreciate you playing enough to notice it.`

Confusion:

`That should be clearer in the UI. I’m improving the first-run explanation so rolling, pressing, banking, and loot are easier to read.`

## Retention Hooks

Day 1:

- first loot card
- first XP bank
- first room clear

Day 3:

- daily Depth Challenge
- new hero recommendation
- “try a greed run” prompt

Day 7:

- unlockable loot card
- best run recap
- new modifier challenge

## Patch Communication

Each patch note should include:

- one player-facing improvement
- one bug fix
- one balance note
- one “next up” teaser

Example:

`Patch 0.1.1 makes first-room combat clearer, improves Shade press feedback, fixes saved loot display, and starts groundwork for animated dice.`
