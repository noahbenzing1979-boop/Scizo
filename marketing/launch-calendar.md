# Launch Calendar

Use this as a relative schedule. Start the clock when the playable mobile build is stable enough to capture screenshots and short clips.

## T-8 Weeks

- Create Google Play app record and reserve the title.
- Start TikTok, Instagram Reels, YouTube Shorts, and Discord branding.
- Post the first five devlog clips:
  - hero select
  - first dice roll
  - first press failure
  - first loot reward
  - before/after visual polish
- Open a private testing list for friends and early supporters.

## T-6 Weeks

- Capture the first full screenshot set.
- Create the feature graphic and icon.
- Draft the long description and short description.
- Build a bug report form.
- Start a weekly `Depth Challenge` format.

## T-4 Weeks

- Enable Google Play pre-registration if launch timing is stable.
- Publish the public store listing.
- Invite 10-20 micro-influencers and small streamers.
- Post two press-your-luck clips and one hero identity clip each week.

## T-2 Weeks

- Freeze the launch feature set.
- Capture final screenshots from the actual build.
- Record the launch trailer.
- Prepare Reddit, Discord, TikTok, and email launch posts.
- Create a patch shortlist for week-one fixes.

## Launch Day

- Publish the game.
- Post the trailer.
- Share the launch post in Discord and relevant communities.
- Ask early players for bug reports, not ratings.
- Track crashes, tutorial friction, and first-session completion.

## Week 1

- Reply to every review.
- Ship one small patch that proves the project is alive.
- Post clutch clips from real runs.
- Start the first public Depth Challenge.

## Week 2

- Run one Play Store experiment.
- Contact the next influencer batch with improved footage.
- Publish a “what players changed” devlog.
- Review retention: first room clear, first loot, first bank, first death, first restart.
